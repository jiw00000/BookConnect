## 3. 백엔드 개발 (Node.js + Express)

### 개요
Node.js와 Express.js를 활용하여 REST API 백엔드 서버를 구축하였습니다.

### 기술 스택

| 항목 | 기술 |
|---|---|
| Runtime | Node.js (v20 LTS) |
| Framework | Express.js v4 |
| DB | MongoDB Atlas + Mongoose (ODM) |
| 인증 | JWT + bcryptjs |
| 파일 업로드 | Multer |
| 환경 변수 | dotenv |
| API 문서 | Swagger (swagger-ui-express) |
| 유효성 검사 | express-validator |
| CORS | cors |
| 로깅 | morgan |
| 개발 서버 | nodemon |

### 구현된 API 엔드포인트

#### Auth `/api/auth`
| Method | Path | 설명 |
|---|---|---|
| POST | `/register` | 회원가입 (학교 이메일 `@ks.ac.kr` 검증) |
| POST | `/login` | 로그인 → JWT 발급 |
| POST | `/logout` | 로그아웃 |
| POST | `/refresh` | 토큰 갱신 |

#### Users `/api/users`
| Method | Path | 설명 |
|---|---|---|
| GET | `/me` | 내 프로필 조회 |
| PUT | `/me` | 프로필 수정 |
| GET | `/` | 사용자 목록 (Admin) |

#### Events `/api/events`
| Method | Path | 설명 |
|---|---|---|
| GET | `/` | 이벤트 목록 |
| GET | `/:id` | 이벤트 상세 |
| POST | `/` | 이벤트 생성 (Admin) |
| PUT | `/:id` | 이벤트 수정 (Admin) |
| DELETE | `/:id` | 이벤트 삭제 (Admin) |
| POST | `/:id/apply` | 이벤트 신청 |

#### Benefits `/api/benefits`
| Method | Path | 설명 |
|---|---|---|
| GET | `/` | 혜택 목록 |
| GET | `/:id` | 혜택 상세 |
| POST | `/:id/claim` | 혜택 수령 |

#### Posts `/api/posts`
| Method | Path | 설명 |
|---|---|---|
| GET | `/` | 게시글 목록 |
| POST | `/` | 게시글 작성 |
| PUT | `/:id` | 게시글 수정 |
| DELETE | `/:id` | 게시글 삭제 |

#### Partners `/api/partners`
| Method | Path | 설명 |
|---|---|---|
| GET | `/` | 파트너 목록 |
| POST | `/` | 파트너 등록 (Admin) |

### 사용자 역할 (Role)
- `student` — 경성대학교 재학생
- `staff` — 교직원
- `admin` — 플랫폼 관리자

### MongoDB Atlas 연결
- MongoDB Atlas 클라우드 DB 연결 설정 완료
- `.env` 파일에 `MONGODB_URI` 환경 변수로 관리

## 4. 전체 프로젝트 흐름 요약

```
기획 → 명세서 작성 → 프론트엔드(Lovable) → 백엔드(Node.js/Express) → MongoDB Atlas 연결
```

1. **기획 단계**: 플랫폼 목적, 대상 사용자, 주요 기능 정의 및 문서화
2. **프론트엔드**: Lovable을 통해 UI 프로토타입 제작 및 배포
3. **백엔드**: Express.js 기반 REST API 서버 구축
4. **DB 연결**: MongoDB Atlas와 Mongoose ODM으로 클라우드 데이터베이스 연동
